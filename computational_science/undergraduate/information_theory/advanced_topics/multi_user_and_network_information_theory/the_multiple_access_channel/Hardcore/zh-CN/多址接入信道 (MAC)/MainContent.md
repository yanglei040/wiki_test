## 引言
在现代通信网络中，从蜂窝系统的上行链路到物联网设备的数据汇聚，多个发送方共享同一无线媒介是普遍现象。这种共享不可避免地带来了信号间的相互干扰，构成了一个核心挑战：如何在多用户共存的环境下实现高效、可靠的通信？[多址接入信道](@entry_id:276364)（Multiple-Access Channel, MAC）模型正是信息论为回答这一根本问题而建立的理论框架，它为我们分析共享信道的容量极限、评估不同接入策略的效率提供了坚实的数学基础。

本文将系统性地引导你深入探索[多址接入信道](@entry_id:276364)的世界。在“**原理与机制**”一章中，我们将揭示定义MAC[容量域](@entry_id:271060)的核心信息论公式，并详细阐述如串行[干扰消除](@entry_id:273045)（SIC）等实现这些理论极限的强大解码技术。接着，在“**应用与跨学科联系**”一章，我们将把理论与实践相结合，探讨MAC模型如何在[无线通信](@entry_id:266253)、资源调度等实际场景中得到应用，并揭示其与[网络信息论](@entry_id:276799)中其他重要概念的深刻联系。最后，通过“**动手实践**”环节，你将有机会亲手计算和分析不同MAC模型的容量，从而巩固所学知识。让我们首先从构成这一切基础的原理与机制开始。

## 原理与机制

在[多用户通信](@entry_id:262688)环境中，多个发送方共享一个公共物理信道，这不可避免地导致信号之间的相互干扰。[多址接入信道](@entry_id:276364) (Multiple-Access Channel, MAC) 模型正是为了研究这一场景下可靠通信的根本极限。本章将深入探讨定义多址信道容量的核心原理，并阐述实现这些理论极限的关键机制。

### 多址[信道容量](@entry_id:143699)域

与单用户点对点信道由单一数值——[信道容量](@entry_id:143699)——来表征其传输极限不同，多址信道的容量由一个**[容量域](@entry_id:271060) (capacity region)** 来刻画。这是一个由所有可实现速率对 $(R_1, R_2, \dots, R_M)$（其中 $M$ 是用户数）构成的集合。对于一个双用户信道，一个速率对 $(R_1, R_2)$ 被认为是**可实现的 (achievable)**，是指存在一套编码和解码方案，使得用户1能够以速率 $R_1$ 发送消息，用户2能够以速率 $R_2$ 发送消息，且在接收端，两条消息可以被同时以任意小的[错误概率](@entry_id:267618)被译出，只要编码块长足够长。

对于一个双用户离散无记忆多址信道，其输入为[相互独立](@entry_id:273670)的[随机变量](@entry_id:195330) $X_1$ 和 $X_2$，输出为 $Y$。由[信道编码定理](@entry_id:140864)可以证明，其[容量域](@entry_id:271060)是所有满足以下不等式的速率对 $(R_1, R_2)$ 的集合，该集合在所有可能的独立输入[分布](@entry_id:182848) $p(x_1)p(x_2)$ 上取并集再求其凸包：

$R_1 \le I(X_1; Y|X_2)$

$R_2 \le I(X_2; Y|X_1)$

$R_1 + R_2 \le I(X_1, X_2; Y)$

这些不等式为我们理解[多用户通信](@entry_id:262688)的内在限制提供了深刻的洞察：

1.  $R_1 \le I(X_1; Y|X_2)$：这个界限给出了用户1的速率上限。其中，[条件互信息](@entry_id:139456) $I(X_1; Y|X_2)$ 表示在接收端**已经完全知道**用户2发送信号 $X_2$ 的情况下，从接收信号 $Y$ 中能够获取到的关于用户1信号 $X_1$ 的信息量。这相当于一个理想化的场景，即用户2的干扰被完美地消除了。
2.  $R_2 \le I(X_2; Y|X_1)$：同理，这是用户2的速率上限，假设用户1的干扰被完美消除。
3.  $R_1 + R_2 \le I(X_1, X_2; Y)$：这个界限被称为**和速率 (sum-rate)** 上限。它指出，两个用户速率的总和不能超过联合输入 $(X_1, X_2)$ 与输出 $Y$ 之间的[互信息](@entry_id:138718)。这代表了信道能够为所有用户可靠传输的总信息通量。

为了将这些抽象的公式具体化，我们来分析一个典型的例子：[二进制加法信道](@entry_id:265650) (Binary Adder Channel) 。在该信道中，两个用户各自发送一个比特 $X_1, X_2 \in \{0, 1\}$，接收端收到的是它们的算术和 $Y = X_1 + X_2$。假设两个用户独立地、等概率地发送0和1，即 $P(X_1=1) = P(X_2=1) = 0.5$。

首先，我们计算和速率的上限 $I(X_1, X_2; Y)$。根据互信息公式 $I(X_1, X_2; Y) = H(Y) - H(Y|X_1, X_2)$。由于输出 $Y$ 是输入 $X_1, X_2$ 的确定性函数，因此给定输入后，输出没有任何不确定性，即 $H(Y|X_1, X_2) = 0$。所以，和速率上限就是输出的熵 $H(Y)$。
输出 $Y$ 的[概率分布](@entry_id:146404)为：
$P(Y=0) = P(X_1=0, X_2=0) = 0.5 \times 0.5 = 0.25$
$P(Y=1) = P(X_1=0, X_2=1) + P(X_1=1, X_2=0) = 0.5 \times 0.5 + 0.5 \times 0.5 = 0.5$
$P(Y=2) = P(X_1=1, X_2=1) = 0.5 \times 0.5 = 0.25$

输出熵为：
$H(Y) = -\sum p(y) \log_2 p(y) = -[0.25 \log_2(0.25) + 0.5 \log_2(0.5) + 0.25 \log_2(0.25)] = 1.5$ 比特。
因此，和速率限制为 $R_1 + R_2 \le 1.5$。

接着，我们计算单个用户的速率上限。对于用户1，其速率上限为 $R_1 \le I(X_1; Y|X_2) = H(Y|X_2) - H(Y|X_1, X_2)$。我们已经知道 $H(Y|X_1, X_2) = 0$。现在计算[条件熵](@entry_id:136761) $H(Y|X_2)$：
当 $X_2=0$ 时，$Y = X_1$。由于 $X_1$ 等概率取 $\{0,1\}$，所以 $Y$ 也等概率取 $\{0,1\}$，此时的[条件熵](@entry_id:136761) $H(Y|X_2=0) = 1$ 比特。
当 $X_2=1$ 时，$Y = 1 + X_1$。由于 $X_1$ 等概率取 $\{0,1\}$，所以 $Y$ 等概率取 $\{1,2\}$，此时的[条件熵](@entry_id:136761) $H(Y|X_2=1) = 1$ 比特。
平均[条件熵](@entry_id:136761)为 $H(Y|X_2) = \sum p(x_2) H(Y|X_2=x_2) = 0.5 \times 1 + 0.5 \times 1 = 1$ 比特。
所以，用户1的速率限制为 $R_1 \le 1$。由于对称性，用户2的速率限制同样为 $R_2 \le 1$。

综上所述，对于[二进制加法信道](@entry_id:265650)和均匀输入[分布](@entry_id:182848)，可实现速率域由以下不等式定义：$R_1 \ge 0$, $R_2 \ge 0$, $R_1 \le 1$, $R_2 \le 1$, 以及 $R_1 + R_2 \le 1.5$。在这个区域内的任何速率对，例如 $(0.6, 0.6)$，都是可实现的，因为 $0.6 \le 1$ 且 $0.6+0.6=1.2 \le 1.5$。而区域外的速率对，如 $(0.8, 0.8)$，则是不可实现的，因为它违反了和速率限制 $0.8+0.8=1.6 \gt 1.5$。

值得注意的是，和速率上限 $I(X_1, X_2; Y)$ 是针对特定的输入[分布](@entry_id:182848) $p(x_1)p(x_2)$ 计算的。在实际应用中，例如两个独立的传感器向一个中心站发送数据，如果它们的[传输概率](@entry_id:137943)是固定的，那么系统的最大总[吞吐量](@entry_id:271802)就由这个特定的输入[分布](@entry_id:182848)决定 。要找到信道的真正[容量域](@entry_id:271060)，需要遍历所有可能的独立输入[分布](@entry_id:182848)，找到使可实现速率域最大化的[分布](@entry_id:182848)。

### 几何特性与简单编码策略

多址[信道容量](@entry_id:143699)域的一个关键数学性质是其**[凸性](@entry_id:138568) (convexity)**。这意味着如果两个速率对 $(R_{1,A}, R_{2,A})$ 和 $(R_{1,B}, R_{2,B})$ 是可实现的，那么连接这两点的线段上的任意一点 $(R_1, R_2) = \alpha(R_{1,A}, R_{2,A}) + (1-\alpha)(R_{1,B}, R_{2,B})$（其中 $0 \le \alpha \le 1$）也是可实现的。

这种[凸性](@entry_id:138568)背后有一个非常直观的物理机制，即**[时分复用](@entry_id:178545) (time-sharing)** 。假设我们有两种可靠的编码方案，方案A能实现速率对 $(R_{1,A}, R_{2,A})$，方案B能实现速率对 $(R_{1,B}, R_{2,B})$。我们可以构造一种[混合策略](@entry_id:145261)：在总传输时间的 $\alpha$ 比例内使用方案A，在剩下的 $1-\alpha$ 比例内使用方案B。在足够长的时间内，用户1的[平均速率](@entry_id:147100)将是 $\alpha R_{1,A} + (1-\alpha)R_{1,B}$，用户2的[平均速率](@entry_id:147100)将是 $\alpha R_{2,A} + (1-\alpha)R_{2,B}$。这就实现了两个已知可实现点之间的任意线性组合。例如，若方案A实现 $(1.50, 0.50)$，方案B实现 $(0.80, 1.80)$，通过以 $\alpha=0.6$ 的比例混合使用，可实现的和速率为 $0.6 \times (1.50+0.50) + 0.4 \times (0.80+1.80) = 2.24$ 比特/信道使用。

最简单的[时分复用](@entry_id:178545)形式是**时分多址 (Time-Division Multiple Access, TDMA)**，即用户在预先分配的、互不重叠的时间片内独占信道。假设用户1单独使用信道时的容量为 $C_1$，用户2单独使用时的容量为 $C_2$。通过分配时间比例 $\alpha$ 给用户1，剩下的 $1-\alpha$ 给用户2，可以实现速率对 $(\alpha C_1, (1-\alpha)C_2)$。所有这些速率对构成了连接 $(C_1, 0)$ 和 $(0, C_2)$ 的一条直线。

考虑一个[高斯多址信道](@entry_id:271906)，两个用户在[加性高斯白噪声](@entry_id:269320)中传输，噪声功率为 $N$ 。用户1的功率为 $P_1$，用户2的功率为 $P_2$。根据香农公式，他们各自独占信道时的容量为 $C_1 = \frac{1}{2}\log_2(1+P_1/N)$ 和 $C_2 = \frac{1}{2}\log_2(1+P_2/N)$。如果目标是实现一个特定的[平均速率](@entry_id:147100)对 $(R_1, R_2)$，我们可以通过求解 $R_1 = \alpha C_1$ 和 $R_2 = (1-\alpha)C_2$ 来确定所需的时间[分配比](@entry_id:183708)例 $\alpha$。

然而，仅仅依靠TDMA这类正交接入方案（即用户在资源上完全分离）是否是最高效的策略呢？答案是否定的。允许用户同时使用信道，并设计巧妙的联合解码方案，通常可以获得更高的总速率。让我们回到[二进制加法信道](@entry_id:265650)的例子 。如果两个用户采用TDMA，当一个用户发送时，另一个用户沉默。此时，信道对于正在发送的用户来说是一个无噪声的二进制信道，其容量为1比特/信道使用。因此 $C_1=C_2=1$。通过[时分复用](@entry_id:178545)，可实现的最大和速率是 $\max(C_1, C_2)=1$。但是，我们之前已经计算过，当两个用户同时发送信号时，和速率上限为1.5比特/信道使用。这 $0.5$ 比特的增益 ($\Delta S = 1.5 - 1 = 0.5$) 表明，允许多用户信号在信道中“混合”，并通过更复杂的接收机技术来分离它们，能够更有效地利用信道资源。这引出了我们下一个关键机制：串行[干扰消除](@entry_id:273045)。

### 串行[干扰消除](@entry_id:273045) (SIC)：一种强大的解码机制

**串行[干扰消除](@entry_id:273045) (Successive Interference Cancellation, SIC)** 是一种强大且实用的多用户解码技术，它能够实现多址信道容量域边界上的某些关键点。其核心思想不是将其他用户的信号视为纯粹的噪声，而是按顺序地、逐个地解码用户信号，并在每一步利用已解码的信息来简化后续的解码任务。

对于一个双用户信道，SIC解码器的流程如下 ：
1.  **选择解码顺序**：接收机首先决定解码哪个用户，比如先解码用户2。
2.  **解码第一个用户**：在解码用户2时，接收机将用户1的信号 $X_1$ 视为额外的随机噪声。此时，对用户2而言，信道是一个单用户信道，其最大可实现速率由其输入 $X_2$ 和输出 $Y$ 之间的互信息决定，即 $R_2 \le I(X_2; Y)$。
3.  **[干扰消除](@entry_id:273045)**：假设用户2的消息被成功解码，接收机就能完美地重建出用户2发送的完整码字序列。然后，它将这个重建信号的贡献从总接收信号中减去。
4.  **解码第二个用户**：在减去用户2的信号后，剩下的信号中不再包含来自用户2的干扰。此时，解码用户1就相当于在一个更“干净”的信道上进行。更准确地说，接收机现在拥有关于 $X_2$ 的[旁路信息](@entry_id:271857)。因此，用户1的可实现速率上限由[条件互信息](@entry_id:139456)给出：$R_1 \le I(X_1; Y|X_2)$。

通过这个过程，SIC解码策略（先解2后解1）可以达到速率对 $(I(X_1; Y|X_2), I(X_2; Y))$。这个速率对恰好是多址信道可实现速率域的一个**角点 (corner point)**。如果反转解码顺序（先解1后解2），则可以达到另一个角点 $(I(X_1; Y), I(X_2; Y|X_1))$。结合[时分复用](@entry_id:178545)，这两个角点以及原点 $(0,0)$ 和两个单用户容量点 $(C_1, 0), (0, C_2)$ 所构成的五边形区域的[凸包](@entry_id:262864)，就完整地定义了整个[容量域](@entry_id:271060)。

让我们将SIC应用于**[高斯多址信道](@entry_id:271906) (Gaussian MAC)**，这是[无线通信](@entry_id:266253)中最重要的模型之一。接收信号为 $Y = X_1 + X_2 + Z$，其中 $X_1, X_2$ 是用户信号，功率分别为 $P_1, P_2$，$Z$ 是功率为 $N$ 的[高斯噪声](@entry_id:260752)。单用户高斯[信道容量公式](@entry_id:267510)为 $C(S/N_{\text{eff}}) = \frac{1}{2}\log_2(1+S/N_{\text{eff}})$。

假设我们采用SIC，并且决定先解码功率更强的用户1（$P_1 > P_2$） 。
*   **第一步：解码用户1**
    此时，用户2的信号 $X_2$ 被视为噪声。因此，用户1的信号功率为 $S=P_1$，而有效噪声功率为 $N_{\text{eff}} = P_2 + N$。用户1的最大可实现速率为：
    $R_1 = C(\frac{P_1}{P_2+N}) = \frac{1}{2}\log_2(1 + \frac{P_1}{P_2+N})$

*   **第二步：解码用户2**
    在成功解码并完美消除用户1的信号后，接收机处理的等效信号为 $Y - X_1 = X_2 + Z$。现在，解码用户2时，信道中不再有来自用户1的干扰。信号功率为 $S=P_2$，噪声功率仅为背景噪声 $N$。此时，用于解码用户2的[信噪比 (SNR)](@entry_id:271861) 就是 $\frac{P_2}{N}$ 。因此，用户2的最大可实现速率为：
    $R_2 = C(\frac{P_2}{N}) = \frac{1}{2}\log_2(1 + \frac{P_2}{N})$

因此，当解码强用户优先时，SIC方案实现了一个角点速率对 $(R_1, R_2) = (\frac{1}{2}\log_2(1 + \frac{P_1}{P_2+N}), \frac{1}{2}\log_2(1 + \frac{P_2}{N}))$。例如，若 $P_1=15$, $P_2=4$, $N=1$，则可实现的速率对为 $(\frac{1}{2}\log_2(1+\frac{15}{4+1}), \frac{1}{2}\log_2(1+\frac{4}{1})) = (1, \frac{1}{2}\log_2(5))$。

SIC的优越性可以通过与一种更简单的策略对比来量化。考虑策略A，即接收机并行地解码两个用户，每次都将对方视为噪声。而策略B是我们刚才分析的SIC策略 。
*   **策略A (并行解码)**：$R_{1,A} = \frac{1}{2}\log_2(1+\frac{P_1}{N+P_2})$, $R_{2,A} = \frac{1}{2}\log_2(1+\frac{P_2}{N+P_1})$。
*   **策略B (SIC)**：$R_{1,B} = \frac{1}{2}\log_2(1+\frac{P_1}{N+P_2})$, $R_{2,B} = \frac{1}{2}\log_2(1+\frac{P_2}{N})$。

两种策略的和速率分别为：
$R_A = \frac{1}{2}\log_2(1+\frac{P_1}{N+P_2}) + \frac{1}{2}\log_2(1+\frac{P_2}{N+P_1})$
$R_B = \frac{1}{2}\log_2(1+\frac{P_1}{N+P_2}) + \frac{1}{2}\log_2(1+\frac{P_2}{N}) = \frac{1}{2}\log_2\left(\frac{N+P_1+P_2}{N+P_2} \cdot \frac{N+P_2}{N}\right) = \frac{1}{2}\log_2\left(1+\frac{P_1+P_2}{N}\right)$

$R_B$ 正是[高斯多址信道](@entry_id:271906)的和速率容量，而 $R_A$ 显然小于 $R_B$，因为 $P_1 > 0$ 导致 $\log_2(1+\frac{P_2}{N}) > \log_2(1+\frac{P_2}{N+P_1})$。通过智能地排序解码过程并消除干扰，SIC能够将一个用户的干扰完全移除，从而显著提升另一用户的速率和系统总[吞吐量](@entry_id:271802)。在一个具体的例子中，当 $P_1=100$, $P_2=25$, $N=5$ 时，SIC策略的总速率可以达到天真策略的约 $1.94$ 倍，这凸显了先进多用户检测技术在现代[通信系统](@entry_id:265921)中的巨大价值。
